# Vibe Design Tutti App Package

This package is generated by `pnpm package:tutti-app` from the `vibe-design` repository.

## Runtime

- `bootstrap.sh` is the only entrypoint Tutti calls.
- The server binds `TUTTI_APP_HOST:TUTTI_APP_PORT`.
- `GET /healthz` is the Tutti healthcheck endpoint.
- Durable data must stay under `TUTTI_APP_DATA_DIR`.
- Runtime scratch files must stay under `TUTTI_APP_RUNTIME_DIR`.
- Logs should go to stdout/stderr or `TUTTI_APP_LOG_DIR`.
- `bootstrap.sh` must launch Node through `TUTTI_APP_NODE`; do not call system `node`, `npm`, or `pnpm` at runtime.

## Package Layout

- `tutti.app.json`: app manifest. The `cli.manifest` field points to `tutti.cli.json`.
- `tutti.cli.json`: command registration for Agent GUI and Tutti CLI integrations.
- `COMMANDS.md`: human-readable command usage for the registered CLI surface.
- `server/dist/main.js`: bundled Node server entry.
- `server/dist/sql-wasm.wasm`: bundled SQLite WASM runtime asset.
- `web/dist`: prebuilt client, stylesheet, and static asset files.
- `skills`: bundled Vibe Design skill catalogue.
- User-imported skills and editable design systems are stored under `TUTTI_APP_DATA_DIR`, not the package root.

## CLI Surface

- Commands are scoped under `vibe-design`.
- Handlers are HTTP `POST /tutti/cli/*` endpoints and return Tutti CLI JSON output.
- Public CLI commands are read-only.
- Project commands expose project list only.
- Conversation commands expose conversation indexes and messages for context lookup.
- Resource commands expose project file resource lists and resource details.
- Comment commands expose preview comment lists.
- Keep `tutti.cli.json`, `COMMANDS.md`, and the server handlers in sync when changing public Agent GUI capabilities.

Do not include `node_modules` in uploaded packages. Runtime dependencies must be bundled into `server/dist` or copied as explicit static assets during the packaging pipeline.
