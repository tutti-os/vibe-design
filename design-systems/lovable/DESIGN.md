# Lovable App Builder Reference

> Category: AI app builder

A playful AI app-builder system inspired by Lovable public pages. Use it for vibe-coding tools, chat-to-app builders, template galleries, prototype workflows, startup landing pages, and idea-to-ship product surfaces.

## Sources

- https://lovable.dev/

The system should feel friendly, creative, and builder-oriented. Use soft pink/purple/blue accents around prompt entry, template discovery, and progress states. Keep the base readable and avoid overwhelming every section with gradients.

## Typography

Use a rounded or friendly sans stack with concise product copy. Headlines can be expressive but should remain legible and direct. Interface text should be plain: build, send, refine, ship, templates, deploy.

Recommended scale:

- Hero: 56-84px desktop, 40-54px mobile, 0.94-1.06 line-height.
- Section title: 32-48px, 1.08-1.18 line-height.
- Template title: 18-24px, 1.16-1.3 line-height.
- Body: 15-18px, 1.5-1.7 line-height.
- Chat/input labels: 13-15px.
- Metric numerals: 40-64px.

## Layout

Lead with a chat-first hero: a short promise, prominent prompt box, send action, and small build mode label. Follow with proof of use, an idea-to-prototype-to-ship explanation, template gallery, builder metrics, and a dense footer for product roles, resources, community, legal, and company links.

Templates should use large thumbnail cards and concise two-line descriptions. Flow sections should be simple and sequential: start with an idea, watch it come to life, refine and ship.

## Components

- Chat hero: prompt card, build mode tag, send button, soft background accent.
- Prompt composer: multiline input, attachment hint, send action, example chips.
- Build progress card: generated pages, connected database, deployment status, preview link.
- Three-step flow: idea, prototype, refine/ship cards with small icons.
- Template gallery: image-first cards, category label, title, short use case.
- Builder metrics: large numbers, short labels, soft divider grid.
- Role links: founders, product managers, designers, marketers, sales, ops, people.
- Footer: resources, product links, legal, community, language selector.

## Interaction

The interface should encourage iteration. Prompt examples should be clickable chips. Template cards can lift slightly and reveal "Start from template." Build progress needs clear generating, previewing, deployed, and error states. CTAs should be rounded and high confidence.

## Boundaries

This package describes AI app-builder structure and visual behavior only. Do not copy Lovable logos, screenshots, template artwork, customer logos, exact metrics, or page copy. Generate original app-builder content while preserving the chat-first creation model, soft creative palette, template gallery rhythm, and idea-to-ship workflow of the reference page.
