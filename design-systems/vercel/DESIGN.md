# Vercel AI Cloud Reference

> Category: Developer platform

A monochrome developer-platform system inspired by Vercel public pages. Use it for cloud platforms, developer tools, AI infrastructure products, docs portals, enterprise SaaS landing pages, deployment dashboards, and security/observability workflows.

## Sources

- https://vercel.com/

## Color

- Canvas: #ffffff

- Panel: #ffffff

- Muted: #666666
- Hairline: #e5e5e5
- Field: #f5f5f5

- Inverted panel: #111111
- Inverted ink: #ffffff
- Success: #0070f3
- New badge: #0070f3
- Warning: #f5a623
- Error: #ee0000

Use black, white, and neutral gray as the core. Blue is a small functional accent for links, new badges, status, and active deployment actions. Avoid decorative color systems, heavy shadows, and gradients unless representing code output or live infrastructure state.

## Typography

Use a crisp developer sans stack and a strong monospace for code, command, metric, and deployment surfaces. Headings should be large, direct, and tightly set. Interface labels should be compact and exact.

Recommended scale:

- Hero: 64-96px desktop, 44-58px mobile, 0.92-1.02 line-height.
- Section title: 36-56px, 1.02-1.12 line-height.
- Product card title: 18-24px, 1.16-1.28 line-height.
- Body: 15-18px, 1.5-1.7 line-height.
- Nav/link labels: 13-15px.
- Code/CLI: 13-15px monospace.
- Metric: 32-56px monospace or tabular numeric.

## Layout

Use a minimal top nav with grouped mega menus for products, resources, solutions, enterprise, pricing, login, and signup. The hero should be centered or constrained, command-oriented, and immediately followed by developer proof: deployment steps, product pillars, metrics, or code panels.

Product navigation is taxonomy-heavy: AI Cloud, Core Platform, Security, Learn, Open Source, Use Cases, and Tools. Footer links should be dense and structured for docs, frameworks, SDKs, community, company, and status.

## Components

- Developer header: compact nav, dropdown groups, Ask AI affordance, login/signup actions.
- Hero CTA row: primary deploy action and secondary demo/expert link.
- Command panel: CLI command, deploy status, build logs, and success indicator.
- Product mega menu: grouped headings, product names, one-line descriptions, small badges.
- Platform pillar card: build, scale, secure, observe, or AI infrastructure block.
- Metric strip: deploy speed, global regions, uptime, latency, or workflow count.
- Link directory footer: multi-column taxonomy with status and theme selector.
- Theme selector: system, light, dark segmented control.

## Interaction

Interactions should be precise. Hover states use subtle gray fills, border contrast, or underline. Menus should open quickly and align to grid columns. Deployment panels need clear pending, building, failed, and ready states. Buttons should feel command-like, not promotional.

## Boundaries

This package describes developer-platform structure and visual behavior only. Do not copy Vercel logos, icons, product illustrations, exact page copy, or proprietary UI screenshots. Generate original developer-platform content while preserving the monochrome design language, product taxonomy, deployment workflow, and documentation density of the reference page.
