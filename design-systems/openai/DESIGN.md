# OpenAI Web Reference

> Category: Editorial application

A restrained black-and-white web system inspired by OpenAI public pages across the homepage, business overview, and API platform. Use it for AI product, developer, research, and enterprise surfaces that need authority without decorative noise.

- Developer dark soft: #1f1f1f

Use monochrome as the default. Green is a functional accent for primary actions, active states, code affordances, and positive system status. Do not build gradient-led pages.

## Typography

Use a neutral grotesk stack. Headings should be large, plain, and confident with tight line-height and normal letter spacing. Body copy should be direct, medium contrast, and generous enough for reading. Avoid decorative display faces.

Recommended scale:

- Hero: 56-84px desktop, 40-52px mobile, 0.94-1.02 line-height.
- Section title: 32-48px, 1.02-1.12 line-height.
- Card title: 18-24px, 1.15-1.25 line-height.
- Body: 15-18px, 1.45-1.65 line-height.
- Meta: 12-14px, compact and muted.
- Code: 13-15px, monospace, dark surface or soft inline block.

## Layout

Use broad whitespace, clear horizontal sections, and an editorial grid. Top navigation is minimal: wordmark text, grouped links, and one or two compact actions. Heroes should be direct, with a large text block and a sparse action row. Dense product information belongs in two-column or three-column grids.

Cards are quiet: low-border, flat or near-flat, with square-to-small radii. Image and media slots should be large, documentary, and directly relevant. If no real media exists, prefer structured panels, lists, or code surfaces over abstract decoration.

## Components

- Header: sticky or top-aligned, minimal links, text wordmark, compact CTA.
- Hero: large title, one paragraph, one primary CTA and one secondary text link.
- Prompt bar: rounded input surface, one-line placeholder, compact submit control.
- Editorial card: image or panel above, category/date meta, title, optional reading time.
- Solution tabs: horizontal segmented control with underlined or filled active item.
- Feature list: short headings, one paragraph, one action link.
- Code panel: dark surface, monospace, high contrast, copy/run affordance.
- Footer: multi-column link groups with small text and clear hierarchy.

## Interaction

Hover states should be subtle: 1px border contrast, soft background shift, or text underline. Buttons use solid ink or green primary states; secondary buttons are outline or text links. Keep motion brief and opacity/translate based.

## Boundaries

This package describes visual structure and tokens only. Do not copy OpenAI logos, illustrations, screenshots, proprietary page copy, or exact marketing layouts. Generate original content while preserving the restraint, hierarchy, and product clarity of the reference pages.
