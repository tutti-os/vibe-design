# Product

## Register

product

## Users

Vibe Design is for builders and designers who use an AI-assisted workspace to create project interfaces, inspect generated design files, and make targeted edits on the canvas without leaving the project editor.

## Product Purpose

The product combines a chat-driven generation flow with a project canvas workspace. Success means a user can describe an interface, open the generated file, inspect the editable nodes, make precise property changes, and keep the design-file state coherent.

## Brand Personality

Focused, practical, craft-aware. The interface should feel like a production design tool: dense enough for repeated editing, clear enough for inspection, and restrained enough that generated work remains the focus.

## Anti-references

Avoid generic form panels, debug-summary surfaces, marketing-page decoration, oversized cards, and any UI copy or identifier that names the external reference product used during visual comparison.

## Design Principles

- Preserve the user workflow before changing presentation.
- Keep editing controls compact, direct, and scan-friendly.
- Prefer real control affordances over explanatory text.
- Make generated design content the center of attention.
- Use neutral product language throughout UI, code, tests, and docs.

## Accessibility & Inclusion

Target keyboard-accessible controls, explicit accessible names, clear focus states, high-contrast text on dark surfaces, and reduced-motion-safe transitions for inspector interactions.
